## 📊 当前系统状态分析

### ✅ 已实现的功能

- **文档管理**
  - ✅ 支持 PDF/Word/TXT 文档上传
  - ✅ 文档列表查看与删除
  - ✅ 文档自动分块与向量化
  - ✅ 基于 Chroma 的向量存储

- **智能问答**
  - ✅ 基于 RAG 的问答功能
  - ✅ 来源文档追溯
  - ✅ 相似度评分展示

- **系统基础**
  - ✅ FastAPI 后端服务
  - ✅ React + TypeScript 前端
  - ✅ 智谱 GLM-4-air + Embedding-3
  - ✅ 健康检查与 API 文档

### ❌ 当前系统的局限

#### **后端**
1. ❌ 没有会话管理：每个查询是独立的，无法保存对话历史
2. ❌ 没有关系数据库：只有向量库，无法存储结构化数据
3. ❌ 没有用户认证：无权限控制和多用户支持
4. ❌ 没有 API 限流：存在滥用风险
5. ❌ 日志未持久化：仅输出到 stdout

#### **前端**
1. ❌ 对话历史不持久化：刷新页面后对话丢失
2. ❌ 没有文档预览：无法查看文档内容
3. ❌ 没有会话管理：无法保存和切换对话
4. ❌ 单轮问答：无法理解上下文（追问）
5. ❌ 缺少导出功能：无法导出对话记录

## 🎯 核心改进目标

基于用户需求和行业最佳实践，本方案聚焦以下核心目标：

1. **多轮对话记录** - 保存完整对话历史，支持会话管理
2. **文档预览功能** - 在线预览上传的文档内容
3. **上下文理解** - 真正的多轮对话能力
4. **用户体验优化** - 流式输出、快捷键、主题切换
5. **企业级功能** - 用户认证、权限管理、监控统计

  ```sql
  CREATE TABLE conversations (

      user_id VARCHAR(36),  -- 预留用户字段
  ```

  ```python
  def build_context_prompt(
      current_question: str,
      history_messages: List[Message],  # 最近 5 轮对话
      retrieved_docs: List[Document]
  ) -> str:
      # 1. 提取历史对话
      history = "\n".join([
          f"{'用户' if msg.role == 'user' else 'AI'}: {msg.content}"
          for msg in history_messages[-10:]  # 最近 5 轮（10条消息）
      ])
      
      # 2. 构建上下文感知的 Prompt
      prompt = f"""
      对话历史：
      {history}
      
      参考文档：
      {retrieved_docs}
      
      当前问题：{current_question}
      
      请基于对话历史和参考文档回答问题。如果问题中有代词（它、这个、那个），请结合对话历史理解其指代。
      """
      return prompt
  ```

#### **5. 流式输出（Streaming）**

**业务价值**：提升用户体验，减少等待焦虑

**后端实现**：
- 🌊 **Server-Sent Events (SSE)**：
  ```python
  from fastapi.responses import StreamingResponse
  import asyncio
  
  @router.post("/conversations/{conversation_id}/stream")
  async def stream_query(conversation_id: str, request: QueryRequest):
      async def event_generator():
          # 1. 检索相关文档
          docs = await vectordb_service.search(request.question)
          yield f"data: {json.dumps({'type': 'sources', 'data': docs})}\n\n"
          
          # 2. 流式调用 LLM
          async for chunk in llm_service.stream_generate(prompt):
              yield f"data: {json.dumps({'type': 'chunk', 'data': chunk})}\n\n"
          
          # 3. 完成标记
          yield f"data: {json.dumps({'type': 'done'})}\n\n"
      
      return StreamingResponse(event_generator(), media_type="text/event-stream")
  ```

**前端实现**：
- 📡 **EventSource API**：
  ```typescript
  const sendMessageStream = async (conversationId: string, content: string) => {
      const eventSource = new EventSource(
          `/api/conversations/${conversationId}/stream?question=${encodeURIComponent(content)}`
      );
      
      let answer = '';
      
      eventSource.addEventListener('message', (event) => {
          const data = JSON.parse(event.data);
          
          switch (data.type) {
              case 'sources':
                  // 显示来源文档
                  break;
              case 'chunk':
                  // 逐字追加答案
                  answer += data.data;
                  setMessages(prev => updateLastMessage(prev, answer));
                  break;
              case 'done':
                  eventSource.close();
                  break;
          }
      });
  };
  ```

- ✨ **打字机效果**：
  ```typescript
  // 平滑的打字机动画
  const TypewriterText: React.FC<{ text: string }> = ({ text }) => {
      const [displayText, setDisplayText] = useState('');
      
      useEffect(() => {
          let index = 0;
          const timer = setInterval(() => {
              if (index < text.length) {
                  setDisplayText(text.slice(0, index + 1));
                  index++;
              } else {
                  clearInterval(timer);
              }
          }, 20); // 20ms 一个字符
          
          return () => clearInterval(timer);
      }, [text]);
      
      return <div>{displayText}</div>;
  };
  ```

  ```python
  class HybridRetriever:
      async def search(self, query: str, top_k: int = 5):
          # 1. 向量检索（语义相似度）
          vector_results = await vectordb_service.search(query, top_k=top_k)
  ```

- 🔄 **使用重排序模型**：
  ```python
  from sentence_transformers import CrossEncoder
  
  reranker = CrossEncoder('BAAI/bge-reranker-large')
  
  def rerank_documents(query: str, documents: List[Document], top_k: int = 3):
      # 计算查询-文档对的相关性分数
      pairs = [(query, doc.content) for doc in documents]
      scores = reranker.predict(pairs)
      
      # 按分数排序
      ranked = sorted(zip(documents, scores), key=lambda x: x[1], reverse=True)
      return [doc for doc, _ in ranked[:top_k]]
  ```

## 🔍 技术难点与解决方案

### **难点 1：大文件预览性能**
**问题**：100MB+ 的 PDF 文件预览卡顿

**解决方案**：
- 分页加载（每次只加载当前页）
- 虚拟滚动（react-window）
- 缩略图缓存
- Web Worker 处理 PDF 解析

### **难点 2：流式输出稳定性**
**问题**：SSE 连接断开、重连逻辑

**解决方案**：
- 实现心跳检测
- 自动重连机制
- 错误处理和降级方案
- 使用 WebSocket 替代（如果需要双向通信）

### **难点 3：多租户数据隔离**
**问题**：确保用户数据安全隔离

**解决方案**：
- 数据库层面：所有表添加 user_id
- 应用层面：中间件自动过滤
- 向量库：为每个用户创建独立 Collection
- 定期审计和权限检查

### **难点 4：对话上下文长度限制**
**问题**：LLM 上下文窗口有限（如 8K tokens）

**解决方案**：
- 只保留最近 N 轮对话
- 使用摘要压缩历史对话
- 分层检索：先检索历史，再检索文档
- 使用长上下文模型（如 GLM-4-128K）
